# Iris Web GUI 高级感与丝滑度提升计划

## 目标

在不破坏当前功能完整性的前提下，把现有 Web GUI 从“功能可用、风格统一”提升到“更高级、更顺滑、更像成熟桌面产品”的状态。

核心关注四件事：

1. **视觉高级感**：层级更清晰、排版更克制、留白更均衡。
2. **交互丝滑感**：切换、展开、发送、加载、保存都更自然。
3. **桌面端体验**：针对 1080p / 2K / 4K 做更稳的密度与宽度策略。
4. **实现可维护性**：把样式从“单点修补”提升为“有体系的 token + 组件规范”。

---

## 当前状态判断

结合当前前端结构（`App.vue`、`AppSidebar.vue`、`ChatView.vue`、`DeployView.vue`、`SettingsPanel.vue`、`main.css`），可以看出：

### 已有优势

- 整体风格已经统一，不是散装页面。
- 组件拆分较清晰，适合继续做精修。
- 已有过渡动画、渐变、玻璃感、暗色/亮色主题基础。
- 聊天、设置、部署三个核心场景已经成型。

### 主要短板

1. **视觉重心略散**
   - 侧栏、主内容、设置面板、部署页都比较“抢眼”。
   - 主次层级有时不够明确，容易显得“都在强调”。

2. **动效缺少统一节奏**
   - 已有动效，但更像“零散存在”，缺少成体系的速度/距离/淡入规则。
   - 某些状态切换还不够“丝滑”，更像直接跳变。

3. **控件密度仍不够产品化**
   - 桌面端虽然已收紧一轮，但还可以更精确地控制：按钮高度、卡片厚度、输入区层级、阴影强度。

4. **反馈细节不足**
   - 请求中、保存中、加载中、复制成功、模型列表拉取成功/失败等反馈虽有，但还可以更高级：更轻、更准、更可感知。

5. **页面间气质不够统一**
   - Chat / Deploy / Settings 虽然都属同一主题，但“同一家产品”的一致性还能更强。

---

## 总体策略

采用“**先体系、后页面；先骨架、后细节；先桌面、后移动**”的方式推进。

分四个阶段进行：

---

## 阶段一：建立视觉与动效基础体系

### 目标

先把全局样式语言定下来，后续所有页面精修都围绕同一套规则展开。

### 要做的事

#### 1. 设计 token 体系再整理

梳理并补足这些变量：

- 尺寸 token
  - 页面最大宽度
  - 内容区最大宽度
  - 侧栏宽度
  - 控件高度（S / M / L）
- 间距 token
  - 4 / 8 / 12 / 16 / 20 / 24 / 32 体系
- 圆角 token
  - 卡片 / 浮层 / 输入框 / pill 分层
- 阴影 token
  - surface / elevated / floating / accent
- 动效 token
  - fast / medium / slow
  - easing 曲线统一
  - hover / expand / route / modal / toast 各类位移幅度统一

#### 2. 明确“信息层级规则”

统一规定：

- 一级标题
- 二级标题
- 区块标题
- 辅助文案
- 状态文案
- 标签 / pill / 注释信息

避免页面之间字号、粗细、字距“各自发挥”。

#### 3. 建立桌面端密度档位

建议建立三档桌面策略：

- **Compact**：适合 1080p
- **Standard**：适合 2K
- **Comfortable**：适合大屏 / 4K

第一步不一定要把切换入口做出来，但样式结构要先支持未来扩展。

### 产出

- 一套前端 GUI 视觉规范草案
- 一套可复用的 CSS 变量 / 规则
- 统一的动效基线

---

## 阶段二：重构 App Shell，让“高级感”先站住

### 目标

先优化整个产品最常见的外层骨架：侧栏、主区域、顶层浮层。

### 要做的事

#### 1. 优化侧栏（`AppSidebar.vue`）

方向：**更轻、更精、更像专业工作台**

重点：

- 缩弱过强的视觉块感
- 降低每个 section 的“体积感”
- 让导航、会话列表、凭证区形成明确层级
- 会话卡 hover / active 状态更细腻
- 删除按钮显隐更自然，不要突兀

可做细节：

- 当前激活会话加入更优雅的左侧高亮条 / 柔和底色
- 新建会话按钮 hover 做轻微 lift + glow，而不是单纯变亮
- 会话项加入更稳的二级信息排版（时间、标题、ID）

#### 2. 优化主内容壳（`App.vue` + `ChatView.vue`）

方向：**减少“满屏铺开”感，提升聚焦感**

重点：

- 主内容区域更加居中
- 对话区与输入区形成更明确的上中下结构
- 页面背景弱化，减少与内容抢注意力
- 让主区域像“精致工作台”，不是“单纯大容器”

#### 3. 优化浮层系统（Settings / Token Dialog）

方向：**统一 modal 气质**

重点：

- overlay 强度
- modal 起落动画
- 标题区与正文区比例
- 关闭按钮 hover 手感
- 滚动区域边界处理

### 产出

- 更稳的整体框架
- 更统一的产品气质
- 页面第一眼观感提升

---

## 阶段三：把聊天体验打磨到“顺滑”

### 目标

聊天页是最常用的场景，要重点打磨“高级感 + 流畅感”。

### 要做的事

#### 1. 优化消息列表（`MessageList.vue` / `MessageBubble.vue` / `ToolBlock.vue`）

方向：**内容更清晰，工具调用更优雅，流式过程更有呼吸感**

重点：

- 用户消息与模型消息层级更鲜明
- 工具块视觉重量下降，不抢正文
- 流式输出中的光标 / 呼吸感更自然
- 代码块、引用块、列表块层级更好
- 图片 / 文档 bubble 更统一

建议：

- 工具块折叠态做成“胶囊摘要”，展开态再显示详细内容
- 正文和工具之间增加更明确但轻量的节奏分隔
- 代码块 toolbar 做更轻的悬浮感

#### 2. 优化输入区（`ChatInput.vue`）

方向：**更像高端聊天产品的 Composer**

重点：

- 输入框高度更稳
- 上传按钮和发送按钮更协调
- 文本输入时有更好的 focus 态
- 拖拽上传时反馈更明确
- 附件条的预览更紧凑有序

可加分项：

- “可发送/不可发送”状态更明确
- Enter / Shift+Enter 提示更自然
- 发送中按钮状态更细腻

#### 3. 优化空状态与过渡状态

方向：**空页面也要有高级感**

重点：

- welcome 区不只是大标题 + 三张卡
- 改成更有节奏的首屏引导
- skeleton / loading / generating 状态统一风格

### 产出

- 聊天页观感升级
- 消息阅读体验更顺
- 输入/发送体验更像成熟产品

---

## 阶段四：统一设置页与部署页的专业感

### 目标

让“设置中心”和“部署生成器”不只是能用，而是看起来真的像产品级控制面板。

### 要做的事

#### 1. 设置中心（`SettingsPanel.vue`）

方向：**更像专业控制台，而不是大表单**

重点：

- tier block 的信息密度更高
- 模型拉取、API Key、Base URL 的关系更清晰
- 成功/失败提示更轻、更近场
- 保存状态与自动保存反馈更高级
- Cloudflare / MCP / LLM 三大区域的区分度更强

建议：

- 对“模型与凭证”做轻量分组标题和操作说明
- 模型列表拉取成功后，加入更轻的成功提示而非只靠字段下说明
- 把过于长的辅助文案折叠成 tooltip / secondary hint

#### 2. 部署页（`DeployView.vue`）

方向：**更像“部署工作台”，减少说明文堆叠感**

重点：

- 左表单右预览更平衡
- 检测卡 / 引导卡 / 结果卡风格统一
- 输出面板代码区域更精致
- 操作按钮的优先级更明确

建议：

- 减少连续大块提示文字造成的视觉压力
- 预览结果区强化“只读工件”的感觉
- 对推荐/警告/错误使用统一状态组件

### 产出

- 设置页更专业
- 部署页更像完整产品能力，而不是工具页拼接

---

## 丝滑感专项（建议贯穿所有阶段）

这是“高级”之外最关键的感觉来源。

### 1. 统一交互动效规范

建议约定：

- hover：1~2px 微抬升
- active：轻微压下
- 展开/收起：180~220ms
- route 切换：180~240ms
- modal：220ms 淡入 + 上浮
- 流式状态：柔和呼吸，不要闪烁过猛

### 2. 加强微反馈

例如：

- 保存成功：短暂静默提示 + 淡出
- 复制成功：按钮局部变色 + 文案瞬时变化
- 拉取模型列表：loading → success / fail 更明显
- 删除会话：按钮 loading / disabled 过渡更自然

### 3. 控制“动效存在感”

原则：

- 不追求炫技
- 让用户感觉顺，而不是感觉“页面在演我”
